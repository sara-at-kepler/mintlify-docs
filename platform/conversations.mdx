title: "Research conversations"

---

Work in Kepler happens in a conversation. You ask a question or request a model, Kepler runs the research, and the result lands in the thread. Follow-ups stay in the same conversation, so Kepler keeps full context from earlier runs.

## Ask a question

Ask in plain language, and be specific about the company and the periods you care about. A ticker or SEC CIK removes any ambiguity about which company you mean.

```text
What drove the change in Costco's gross margin over the last three fiscal years?
```

## Watch the run

Kepler finds the relevant filings and transcripts, extracts the figures, and writes the answer. A run takes a few minutes, and you see live progress as it reads sources, so you can follow what it is working from.

## Read the result

Every figure in the answer links back to the filing or transcript it came from. Open a citation to see the original source, and review the [sourcing report](/platform/sources) to confirm coverage before you rely on a number. Model requests also return a [workbook](/platform/workbooks).

## Follow up in the same thread

You do not need to start over to extend your work. Ask a follow-up and Kepler keeps the context from earlier runs:

```text
Now compare that to Walmart for the same period.
```

Add a column to a model, break out a segment, or bring in a peer, all as follow-ups. Start a new conversation only when you switch topics. Your conversations are saved to your account, so you can revisit an earlier run and pick it up later.

<Tip>
  The same conversations are reachable through the [MCP connector](/mcp/overview). A run you start in your assistant shows up here, and one you start here can be continued from your assistant.
</Tip>
